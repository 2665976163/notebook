通过eclipse git工具合并多个分支

百度经验:jingyan.baidu.com

实际开发中往往多个分支同时开发，最终合并到master主分支上，我创建了一个dev分支和一个other分支，分支上版本做了修改以后合并到master分支上。

百度经验:jingyan.baidu.com

工具/原料

  • eclipse
  • git
百度经验:jingyan.baidu.com

方法/步骤

  1. 1

    首先我们把主分支和other分支合并，切换到主分支

    步骤阅读
  2. 2

    第二步右击team -> merge

    步骤阅读
  3. 3

    弹出merge框以后个人建议首先本地合并然后push到远程，选择local分支里的合并目标分支other，点merge。

    步骤阅读
  4. 4

    这次合并不会出现冲突，我们首先把master分支提交到远程，other分支的合并完成

    步骤阅读
  5. 5

    合并master分支和dev分支，这个过程会出现分支，我们要保证other分支提交的内容不丢失。第一步切换到主分支，有些小伙伴切换分支，出现如图的框是因为你修改的内容没有提交可以提交本地仓库也可以提交到远程仓库，我们先提交到本地（没出现进入下一步）

    步骤阅读
  6. 6

    我们右击选择merge，merge框里选择我们合并的分支

    步骤阅读
  7. 7

    merge以后出现冲突文件了，我们首先处理冲突文件，先把冲突符号删除，只留我们想要的代码。要注意master分支和dev的重复代码。冲突处理一般都要手动处理。

    步骤阅读
  8. 8

    处理完冲突以后，add to index 以后提交到远程就可以了，合并完成了

    步骤阅读
    END
百度经验:jingyan.baidu.com

注意事项

  • 注意合并过程中代码必须提交到仓库
